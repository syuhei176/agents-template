# エージェント仕様書の書き方

このドキュメントは、カスタムエージェントの仕様書を作成する際のガイドラインです。

## 仕様書の構成

エージェント仕様書は以下の構成で記述してください：

### 1. 概要セクション

```markdown
# エージェント名

エージェントの目的と役割を簡潔に説明します。
```

### 2. 責任範囲

エージェントが担当する具体的なタスクや責任範囲を明確に定義します。

```markdown
## 責任範囲

- タスク1の説明
- タスク2の説明
- タスク3の説明
```

### 3. 入力と出力

エージェントが受け取る入力と生成する出力の形式を定義します。

```markdown
## 入力

- 入力パラメータ1: 説明
- 入力パラメータ2: 説明

## 出力

- 出力形式の説明
- 期待される結果
```

### 4. 制約事項

エージェントの動作における制約や注意事項を記載します。

```markdown
## 制約事項

- 制約1
- 制約2
- 制約3
```

### 5. 実行例

具体的な使用例を示します。

```markdown
## 実行例

### 例1: タスク名

入力:
\```
入力例
\```

期待される出力:
\```
出力例
\```
```

## 詳細仕様の書き方

詳細仕様は以下の観点で記述してください：

### アルゴリズムとロジック

- エージェントの内部処理フロー
- 判断基準とロジック
- エラーハンドリング

### パフォーマンス要件

- 実行時間の目標
- リソース使用量の制限

### 品質基準

- 成功の定義
- テスト方法
- 評価指標

## ベストプラクティス

1. **明確さ**: 曖昧な表現を避け、具体的に記述する
2. **完全性**: 必要な情報をすべて含める
3. **一貫性**: 用語や表記を統一する
4. **更新性**: 変更履歴を記録する
5. **テスト可能性**: 検証可能な基準を設定する

## テンプレート

以下は基本的なテンプレートです：

```markdown
# [エージェント名]

## 概要
[エージェントの目的と役割]

## 責任範囲
- [責任1]
- [責任2]

## 入力
- [パラメータ1]: [説明]

## 出力
- [出力形式]

## 制約事項
- [制約1]

## 実行例
### 例1
入力: [入力例]
出力: [出力例]

## 詳細仕様
### 処理フロー
1. [ステップ1]
2. [ステップ2]

### エラーハンドリング
- [エラーケース1]: [対応方法]
```

## 更新手順

仕様を更新する際は：

1. このドキュメント（specs/AGENTS.md）を参照
2. 上記のガイドラインに従って仕様を記述
3. 変更内容を記録
4. レビューを受ける
